4.继承（extends，super）

p150

格式：public class 子类名 extends 父类名（）

例如：public class Zi extends Fu()

特点：子类有父类的内容，父类无子类内容

public class Zi extends Fu{
    public int age=20;
    public void show(){
        int age=30;
        System.out.println(age);//访问局部变量age
        System.out.println(this.age);//访问成员变量age（全局变量）（本类的成员变量）
        System.out.println(super.age);//访问父类的成员变量
    }
}

子类构造方法

子类中所有的构造方法默认都会访问父类中无参的构造方法

  1. 因为子类会继承父类中的数据，可能还会使用父类的数据。所以，子类初始化之前，一定要先完成父类数据的初始化。
  2. 每一个子类构造方法的第一条语句默认 super（）；
  3. 如果父类没有无参构造，推荐手动给每一个类添加一个无参构造方法；

解释如下代码

public class Fu{
    public Fu(){
        System.out.println("fu中无参被调用");
    }
}
public class Zi{
    public Zi(){
        System.out.println("Zi中无参被调用");
    }
    public Zi(int age){
        System.out.println("Zi中有参被调用");
    }
}
public static void main(String[] agre){
    Zi z=new Zi();
    //输出：fu中无参被调用，zi中无参被调用
    Zi z2=new Zi(20);
    //输出：fu中无参被调用，zi中有参被调用
}

6.final

p165

特点：

  1. 修饰”方法“：表明该方法是最终方法，不能被重写
  2. 修饰变量：表面该变量是“常量”，不能再次被赋值
  3. 修饰“类”：表明该类是最终类，不能被继承（子extends父）

final修饰局部变量

  1. 变量是基本类型：final修饰的是基本类型的数据值不能改变
  2. 变量是引用类型：final修饰的是引用类型的地址值不能改变，但地址值可以变
final int a=10;//基本类型
final Student s=new Student();//引用类型
public final class Fu{//类
    public final void method(){//方法
    }
}

### 类和接口的关系

1. 类和类的关系：继承（extends）关系，只能单继承，但可以多层继承

2. ```java

  4. 类和接口：实现（implements）关系，可以单实现也可以多实现，可以在继承类的同时实现多个接口

  5. public class Zi extends Fu implements jie1,jie2,jie3{}

5. 接口与接口：继承（extends）关系，可以单继承，也可以多继承

#### 抽象类与接口的区别

抽象类是对事物（门）的抽象（仍是那类事物共有的特性）

接口是对行为（报警）的抽象（额外的，特有的）

例如：门具有开和关的功能，采用抽象类，但加了个报警功能，则用接口来实现

21.Lambda

p355

注意

  1. 使用Lambda，接口只能有一个抽象方法
  2. 只有接口才能用Lambda

格式

(形参)->代码块

  1. 形参：多个参数时逗号隔开即可
  2. ->：指向动作
  3. 代码块：以前写的方法体内容

例如：

//匿名内部类
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello World");
    }
}).start();
//Lambda表达式
new Thread(() -> {
    System.out.println("Hello World");
}).start();
  1. 形参的类型可以省略
  2. 如果参数只有一个，形参的小括号可以省略
  3. 如果代码块只有一条语句，大括号和分号也可以省略，如果语句有return，return也省略掉

接口的默认方法

用处：在接口功能不能满足用户需求时，又要考虑代码的维护，而新建一个默认方法，以便于不需要给所有实现该接口的方法重写。

注意：你不能在实现两个接口的同时，在不同接口写相同名字的默认方法，会报错

如果一个类直接实现了两个接口，并且这两个接口都定义了相同名称的默认方法，那么编译器会报错，因为它不知道应该使用哪个接口的默认实现。
